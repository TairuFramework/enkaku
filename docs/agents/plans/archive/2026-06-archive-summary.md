# 2026-06 Archive Summary

## Plans Completed

- **inspect-group-info** (2026-06-03, complete) -- Added `inspectGroupInfo(groupInfoBytes)` and a `GroupHandle.treeHash` getter to `@enkaku/group` (0.16.2), letting a stale device non-destructively confirm its external-resync Commit was canonically accepted. Decodes framed GroupInfo, returns `{ epoch, treeHash }` off `groupContext`; throws on malformed/wrong input (contrast with never-throwing `readMessageEpoch`). No signature verification (structural read of trusted bytes). Follow-on to mls-external-rejoin.

- **mokei-otel-schema-upstream** (2026-06-09, complete) -- Two upstream asks for mokei MCP draft-migration, shipped together on `feat/mokei-requirements`. **G5:** `@enkaku/otel` gained `tracestate` + `baggage` codecs (`formatTracestate`/`parseTracestate`, `formatBaggage`/`parseBaggage`) completing the W3C Trace Context propagation trio. **G8:** `@enkaku/schema` support for arbitrary JSON Schema 2020-12 for tool schemas. Both were blocking mokei.

- **enkaku-mokei-downstream-asks** (2026-06-12, complete) -- Closed four API-surface gaps for downstream kubun/mokei on `chore/mokei-follow-ups` (consolidates two ask-docs). Headline: `@enkaku/server` now exports `HandlerError` + `HandlerErrorParams`; `executeHandler`'s `from()` short-circuits on `instanceof` so a custom `code`/`message` survives to the client instead of collapsing to the generic envelope (unblocks kubun's wire-distinguishable `SyncAccessDeniedError`/`KB08`). Plus otel baggage + schema-strict enablers. Publishing/version bumps owned by maintainer; post-publish verification extracted to `next/`.

- **transport-framing-limits** (2026-06-12, complete) -- Gave every untrusted-stream-ingesting Enkaku transport a bounded-memory knob and closed an unbounded-memory gap in the JSON-lines framer (PR #36, from mokei hang/crash-core audit item 6). `@enkaku/stream` extracted a shared `FramingLimits` base; `fromJSONLines` `maxBufferSize` now bounds total framer memory. Wired into `@enkaku/node-streams-transport`, `@enkaku/http-client-transport` (eventsource-parser cap + abort on overflow), and `@enkaku/http-server-transport` (new `maxRequestBodySize`, default 1 MiB, → 413 on oversize).

- **channel-send-missing-prc** (2026-06-14, complete) -- Fixed silent data loss on validating servers (`fix/hub-send`). `Client.createChannel().send()` omitted `prc` from its payload; the per-procedure send schema requires it, so every real-client channel `send` failed validation and was silently dropped (the validation-failure branch never emitted errors for `send`). Most impactful: hub `hub/receive` acks were discarded, so kubun mailboxes never drained. Latent since v0.8 (Dec 2024) — a docs consistency pass added `prc` to the schema but never to the client builder. Now includes `prc` and stops dropping invalid sends silently.

- **fromemitter-return-settles-pending** (2026-06-15, complete) -- Stopped `@enkaku/generator`'s `fromEmitter` from hanging a parked `next()` forever on close. `pending` reshaped to `Deferred<IteratorResult>`; the single `stop()` choke point (called by `return()`/`throw()`/dispose/abort) now resolves any outstanding `pending` with `{done:true}`. Also: falsy events (`null`/`undefined`) now delivered instead of dropped; pre-aborted signal guarded. Sibling of the kubun `consume()` teardown fix.

- **kubun-groupcontext-extension-support** (2026-06-18, complete) -- Let `@enkaku/group` consumers create joinable MLS groups carrying a custom GroupContext extension (kubun's genesis anchor: the epoch-0 admin DID) and reject commits that mutate it. Pure plumbing of ts-mls primitives enkaku already depends on; the commit hook exposes the raw ts-mls `IncomingMessageCallback` rather than a wrapper.

- **fromstream-cancel-on-return** (2026-06-19, complete) -- Made `@enkaku/generator`'s `fromStream` cancel the underlying `ReadableStream` on early return (`fix/fromstream-cancel`), matching native async-iterator semantics instead of only releasing the reader lock. `finally` now runs `await reader.cancel().catch(() => {})` before `releaseLock()`, so the source's `cancel()` fires and releases HTTP connections / child pipes / timers. `preventCancel: true` preserves old behavior.

- **group-messaging-primitives** (2026-06-20, complete) -- Added group-messaging fan-out primitives so a group is a first-class messaging substrate (PR #41, 47 commits). Three independent layers: **Phase 1** `@enkaku/broadcast` (new) — generic fan-out, no MLS/hub/DID coupling; **Phase 2** hub pub/sub — reworked `@enkaku/hub-protocol`/`-server`/`-tunnel`/`-client` into a blind, capability-gated broker; **Phase 3** `@enkaku/group-rpc` (new) — directed 1:1 RPC to a single member. All over epoch-rotating opaque topics with an authenticated sender, hub kept blind to membership.

- **otel-w3c-inbound** (2026-06-20, complete) -- Server-side W3C trace propagation for `@enkaku/otel` (mokei G5 inbound, PR #42). Three exports, each the inverse of an existing one, reusing existing parse primitives: `entriesToBaggage` (inverse of `baggageToEntries`, lossless round-trip), `extractW3CTraceContext` (traceparent counterpart to `extractTraceContext`, returns `undefined` when tracing off), and `withActiveBaggage` (activates baggage for a callback). Lets a server consume client trace context without importing `@opentelemetry/api` directly.

- **stack-refactoring** (2026-06-30, complete) -- Retooled `enkaku` onto the five-repo Yulsi split (`@kigu`/`@sozai`/`@kokuin`/`@kumiai`) and trimmed it to an RPC-only repo (PR #45, branch `chore/stack-refactoring`, 25 commits). Four sequenced workstreams including the v0.18 migration phase 1: tooling retooled onto `@kigu/dev` (root `biome.json` extends it, swc/tsconfig point at it, changesets added, toolchain devDeps dropped to transitive), RPC published at 0.18.0. Parent design lives in `kigu/docs/repo-split-design.md`.
